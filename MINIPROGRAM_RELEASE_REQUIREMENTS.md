# 微信小程序发布要求清单

本文件是工程准备清单，不是法律意见。金融、AI 和个人信息规则可能变化，正式提交前必须在微信公众平台后台核对当时可选类目、资质和审核要求。

## 体验版

- 注册小程序并取得 AppID；课程阶段也可用开发者工具测试 AppID。
- 将 `miniprogram_prototype/` 导入微信开发者工具。
- 保持 `demoMode: true` 可完成无需网络的课堂展示。
- 不把“体验版可运行”描述成“已公开上架”。

## 连接后端

- 使用有效 HTTPS 域名，不在前端关闭域名校验作为发布方案。
- 在小程序后台配置 request 合法域名。
- 后端增加用户鉴权、CORS/来源约束、速率限制、审计日志和稳定 `/v1` 契约。
- 所有 API Key、Token、AppSecret 和数据库管理密钥只放在服务器端。

## 备案与审核

- 小程序应通过平台完成应用备案流程；工信部说明移动互联网应用需要履行备案，平台负责协助小程序备案。[工信部政策解读](https://www.miit.gov.cn/zwgk/zcjd/art/2023/art_39b4f1acc36745b98478e0ec3e07128d.html)
- 提交前核对主体类型、服务类目、名称、功能描述、隐私指引和可能需要的资质。
- 产品描述使用“交易前个人规则检查、投资者教育、信息整理”，不得伪装实际功能，也不得宣传荐股、保证收益或自动交易。
- 涉及金融信息服务的类目与资质判断必须以提交时后台结果和专业合规意见为准；不要把课程原型的免责声明当成资质替代品。

## 隐私

- 隐私保护指引必须与实际收集和调用完全一致。
- 在调用涉及个人信息的接口前完成必要告知和同意流程；腾讯云对微信要求的说明指出，涉及个人信息时需同步用户已阅读并同意隐私规则。[腾讯云适配说明](https://cloud.tencent.com/document/product/1301/97930)
- 默认不申请手机号、位置、通讯录、相册、麦克风或微信昵称头像权限。
- 如果后续启用语音，先更新数据地图和隐私指引，解释音频是否上传、是否保存及使用哪家识别服务。
- 提供查看、导出、删除和撤回同意的入口。

## 上线前测试

- 模拟 API 超时、重复提交、无网络、结构化输出失败和知识库无结果。
- 检查用户能更正规则和理由分类。
- 检查“未找到支持”不会被展示成“消息为假”。
- 检查自伤表达会停止股票分析、显示真人帮助资源，并且不保存该段审查。
- 对代码包做密钥扫描，确认没有 AppSecret、OpenAI Key、股票 Token 或数据库密码。
