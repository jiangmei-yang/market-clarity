# 移动端 UX 审查

审查日期：2026-07-21  
页面：首页、个股、自选、持仓、交易日志、设置、数据隐私  
测试尺寸：360×800、390×844、412×915

基线截图：`docs/screenshots/mobile_baseline/`  
改进后截图：`docs/screenshots/mobile_final/`

最终验收截图使用 `*_evidence.png` 文件；三张首页分别对应目标尺寸，另有 `stock_390x844_evidence.png` 展示个股首屏。

## 测试说明

在 Codex 内置浏览器中设置三个目标 viewport。该浏览器为保持渲染可读性会报告约 450/487/515 CSS 像素的内部宽度，因此验收同时检查“文档宽度是否大于实际 viewport”、按钮几何高度和真实截图，而不只看输入尺寸数字。

## Baseline findings

### P0

1. **移动导航不可用。** Streamlit 自动侧边栏展开时几乎占满窄屏，页面正文只能从边缘看到；主要任务不在拇指区域。
2. **点击区域太小。** 多个可见按钮测得约 18–31px 高，低于建议的约 44–48px 触摸区域。
3. **个股页首屏选择过多。** 时间范围、技术指标和长图表抢在主要结论前后，用户需要大量滚动。
4. **“AI/产品说明式”文案。** 句子抽象、完整但不口语，例如“决策前检查”“固定数据来源和规则证据”，中老年用户不容易快速抓住动作。

### P1

1. 首页三栏价值说明在手机上纵向拉长，核心任务后出现大量解释。
2. 指标图默认同时显示 RSI 与 MACD，图高、图例多，标签拥挤。
3. 宽表格在已有数据时只适合横向滑动，不适合快速扫读。
4. 交易日志、设置、数据隐私与核心页面在一级导航同权重。
5. 部分内容依赖 Plotly 悬停才能看到精确数值；触屏不稳定。

### P2

1. 默认字体约 15.75px，正文尚可但辅助文字偏小。
2. Streamlit 顶部工具栏和页面切换动画不像原生 App。
3. 图表工具栏图标在小屏较密。

## Changes completed

### P0

- 手机宽度隐藏侧边栏，加入固定底部导航：首页、个股、自选、持仓、我的。
- 所有主要按钮和输入框最小高度 48px。
- 手机端多栏自动改为单栏；页面底部预留安全区域。
- 首页文案改成更口语的“查一只股票，先看看哪里要小心”。
- 个股页默认不打开 RSI/MACD/布林带；先显示 5 日与 20 日变化文字。

### P1

- 首页持仓由宽表格改为卡片。
- 新建“我的”页面，把交易记录、设置、隐私和添加到桌面说明集中起来。
- 次要说明和财务历史放入折叠区域。
- 图表之外增加文字结论，不要求用户悬停才能理解。

## Remaining issues

### P1

- Streamlit 的 Plotly 图表仍比专用移动前端高，图例和工具栏只能有限控制。
- 自动生成的侧边页面仍会出现在桌面导航；手机端已隐藏并改用底栏。
- Streamlit 内置 dataframe 仍是横向滚动组件；核心路径已改卡片，但后台明细页面还可能使用表格。

### P2

- 底部导航是响应式 HTML 导航，不具备原生 App 的手势、转场和系统返回栈精细控制。
- 浏览器“添加到主屏幕”的外观因 iOS/Android 和浏览器版本而异。

## User-test gates

- 10 秒内无需帮助找到股票输入框：目标 ≥4/5。
- 20 秒内找到“一分钟看懂”和第一条风险：目标 ≥4/5。
- 单手从首页进入个股、自选、持仓、我的：目标 5/5。
- 无人把演示数据认成真实行情。
