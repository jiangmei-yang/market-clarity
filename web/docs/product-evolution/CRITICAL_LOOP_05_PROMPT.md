# Critical Loop 05 执行提示词

你现在不是继续增加安心看股功能，而是执行真实验证。当前可辩护课程分为 90/100；在没有外部证据前，不得将分数上调。

## P0：完成 15 位外部用户测试

1. 严格按 `REAL_VALIDATION_RUNBOOK.md` 执行。
2. 每位用户使用唯一匿名编号，自己提交反馈。
3. 任务开始、完成、放弃和候补曝光由系统记录；不手工修改原始数据。
4. 每 5 人为一组复盘：只修最频繁、最严重的一个卡点，然后继续测试。

## P0：运行真实模型评测

1. 连接一个真实 Provider，记录模型名称、Provider、Prompt 版本和时间。
2. 在 `/evaluation` 运行 20 项固定任务。
3. 保留失败原文；修正后创建新版本，不覆盖旧结果。
4. 在课堂只展示真实模型得分，规则模式只是安全基线。

## P0：获得行为型商业证据

1. 每位参与者都看到同一个价格假设：14 天后 19 元/月。
2. 只有点击“加入测试”才计入候补；反馈表的态度题不算转化。
3. 报告独立用户曝光、加入、退出和转化率。
4. 少于 5 人加入时，不改口径；访谈不加入原因。

## 停止条件

只有同时满足以下条件，才可以请评委按 95+ 分复评：

- 外部用户样本 ≥ 15。
- 任务完成率 ≥ 80%，且快速结束已单独报告。
- 风险理解 ≥ 60%。
- 有至少一个可讲述的真实行为改变案例。
- 候补加入 ≥ 5。
- 真实模型固定任务结果已保存。
- 不存在任何模拟用户或 Mock 结果被当作真实证据。
