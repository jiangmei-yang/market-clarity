# Critical Loop 12：版本统一与 Agent 三角色复测

日期：2026-07-23

## 评委批评

第 11 轮虽然已经在浏览器中完成三类真实任务，但证据仍依赖一次性的人工操作。更严重的是，GitHub `main` 是包含后端与 `web/` 的主仓库，而 Sites 候选分支把前端放在仓库根目录；两者没有共同提交历史。直接强推会让组员失去 FastAPI、数据处理和部署结构。

## 本轮修改

1. 以 GitHub `main` 为主线，保留根目录后端，把验证后的前端同步到 `web/`；
2. 主线生产构建和全部测试通过后才推送；
3. 新增 `/api/evaluation/agent`；
4. 新增三类固定 Agent 功能任务和逐项评分；
5. 新手任务的工作台修改只预览，评测后自动取消；
6. 持仓任务使用当前用户保存的真实持仓，不伪造行业；
7. 个股任务真实调用行情、财务和公告服务，并核对来源与时间；
8. 评测页同时显示工具数、来源数、模型和失败证据。
