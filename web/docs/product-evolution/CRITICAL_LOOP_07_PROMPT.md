# Critical Loop 07 执行提示词

禁止继续增加产品模块。部署第六轮验证修复后，唯一任务是获得第一批 5 位外部用户的未经美化证据。

## 操作

1. 招募 `投资经验不足 1 年` 用户 2 位、`ETF 或长期持有` 用户 2 位、`近 3 个月主动交易` 用户 1 位。
2. 给每人一个匿名编号，不收姓名、微信或真实资产。
3. 按 `REAL_VALIDATION_RUNBOOK.md` 完成同一任务，不提示按钮位置。
4. 保留所有快速退出、放弃、负面满意度和“不愿再次使用”。
5. 两位团队成员独立编码风险复述，保留分歧。
6. 只选择出现频率最高、最阻碍完成的一个问题进行修改。
7. 修改前导出会话和反馈 CSV，修改后再测下一批，禁止覆盖前一批。

## 本轮通过条件

- 外部用户 ≥ 5；
- 三类用户都有覆盖；
- 每位用户均明确同意匿名研究；
- 完成率、快速结束、放弃、风险复述和再次使用均有真实分母；
- 至少记录 1 条负面或困惑反馈；如果没有，必须说明招募偏差而不是宣称完美。

完成第一批 5 人只能复评产品问题，不能自动加到 95 分。总样本达到 15、真实模型固定题集完成且价格行为证据达标后，才进行 95 分审计。
