# Critical Loop 03 执行提示词

你正在 `/Users/nu/Documents/Stock/sites_frontend` 中继续改进“安心看股”。不要新增泛功能。目标是把课程评分从当前可辩护的 85 分推进到有真实证据支持的 90+，并明确哪些 95 分门槛必须由真实用户完成。

## P0：真实模型评测

1. 在 `/evaluation` 增加“运行真实模型评测”动作，仅在非 Mock Provider 已连接时启用。
2. 固定至少 20 个任务，覆盖：拒绝荐股、数据缺失诚实度、来源引用、结构化解析、社交跟风识别、不得承诺收益。
3. 保存 provider、model、prompt version、运行时间、原始输出、评分规则和失败项。
4. 模型不可用时明确显示未运行；不得生成模拟分数。
5. 评测失败不能影响确定性规则功能。

## P0：真实用户验证闭环

1. 建立匿名测试会话，记录开始、完成、放弃、耗时和最终选择。
2. 核心任务固定为：粘贴一条投资说法 → 核实来源 → 查看持仓影响 → 完成交易前检查 → 维持、修改或延迟。
3. 导出匿名 CSV，零用户时显示 0；不得预置模拟用户。
4. 评测中心只汇总真实记录，并清楚标注样本量。

## P0：90 秒演示

1. 建立一个“课堂演示”入口，使用明确标注日期和来源的教学快照。
2. 演示不依赖外部接口和真实模型，但必须清楚标注“教学快照 / 规则解释”。
3. 流程必须在 90 秒内完成，并展示用户修改或延迟计划前后的数字变化。

## P1：商业验证

1. 只测试一个付费假设：每月持续监控持仓判断变化与风险提醒。
2. 使用真实行动指标：加入候补、留下联系方式或进入付费测试，而不只问态度题。
3. 明确免费与付费边界、数据成本和服务频率。

## 验收

- 所有新增结果可追溯，不用 Mock 冒充模型或用户证据。
- 外部服务失败时核心规则流程仍可完成。
- `npm test` 全部通过。
- 重新以课程五项权重评分，并给出原始证据链接或文件路径。
- 没有真实证据时不得宣布 95 分。
