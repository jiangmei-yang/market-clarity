# Critical Loop 12：版本统一与 Agent 功能证据

你正在维护 Market Clarity｜安心看股。不要新增与核心任务无关的金融模块。

本轮目标：

1. 审计 GitHub `main`、前端候选分支与 Sites 候选版本，确保组员看到的是同一套可运行代码；
2. 保留仓库根目录的 FastAPI、数据与部署资产，把已验证前端同步到 `web/`，禁止用无关历史强推覆盖；
3. 建立三类可重复 Agent 功能任务：新手工作台、真实持仓风险和 600519 行情/财务/公告研究；
4. 评测必须调用当前已连接的真实 Provider；只有规则模式时阻断，不得冒充生成式 Agent；
5. 所有未确认工作台修改在评测后取消，不能污染用户配置；
6. 输出每项检查、实际证据、工具数、来源数、Provider、模型和运行时间；
7. 明确区分工程功能任务、模型安全题集、数据路由采样、外部用户与付费证据。

完成标准：

- GitHub `main` 保留原有后端和仓库结构；
- 前端生产构建和全部自动测试通过；
- 三类真实 Agent 任务可以重复运行；
- 评测结果失败时原样显示；
- 不把内部评测写成用户满意度、留存或收入；
- 没有外部样本时继续显示 0，不得把课程总分写成 95。
