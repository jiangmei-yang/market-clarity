# Critical Loop 11：从“会规划”到“能完成”

日期：2026-07-23

## 评委批评

上一版 Agent 能生成漂亮的步骤面板，但真实任务暴露了三个 P0：

1. 新手只要求安排工作台，模型却加入提醒和观察列表，形成未请求的确认动作；
2. 持仓风险工具只显示“组合金额 80,000，持仓 1 项”，没有回答最大仓位和行业风险；
3. 股票研究列出了行情、财报和公告步骤，却全部返回“去对应页面执行”，属于占位工具而不是 Agent。

这类问题会让评委认为产品只是把大模型规划文本套在静态页面上。

## 本轮修改

- 模型输出降级为建议：确认型工具只有在用户原话明确要求时才能进入执行计划；
- 移除模型臆造工具导致的虚假“能力缺口”；
- 新手工作台不再擅自创建提醒或观察列表；
- 组合风险返回最大持仓、占比、个人阈值、行业暴露、行业缺失和待核对项；
- 缺少行业标签时不再把“待核对”当成一个行业并计算超限；
- 行情工具接入平台真实公开行情和缓存/降级控制；
- 财务工具接入公开三表体检与确定性异常规则；
- 公告工具接入近 90 天公开披露检索，并保留原文链接；
- 研究结果新增“先看这几项”，把价格、财务异常和公告原文核对连接起来；
- 数据来源区显示实际来源和数据时间，不再只写泛化 Registry 名称。

## 仍未完成

- 当前 Agent 结果摘要是确定性工具整理，不是对所有开放问题都能生成高质量综合研究报告；
- 公告备用源是聚合入口，仍需用户打开法定披露原文；
- 当前持仓只有 1 项且缺行业标签，不能证明多资产行业聚合已被真实用户理解；
- 外部参与者仍为 0，不能把本轮模拟当成用户验证；
- 本轮候选版本尚未发布。

## 评分影响

本轮修复的是“可运行 MVP”的实质能力，而不是新增导航入口。已部署版本仍为 `82/100`；本地候选版本从 `83/100` 调整为 `85/100`。距离课程 95 分的主要差距仍是外部用户行为和付费证据，而不是继续增加工具数量。
