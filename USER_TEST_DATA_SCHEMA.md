# 匿名用户测试数据字段

测试只使用模拟资金和模拟持仓。测试编号使用 `T001` 等匿名编码，不建立姓名、手机号或账户映射。

| 字段 | 类型 | 用途 |
|---|---|---|
| tester_code | 文本 | 匿名区分测试场次 |
| review_id | 整数/空 | 关联本次本地审查记录 |
| rules_completed | 布尔 | 是否完成个人规则设置 |
| original_plan_json | JSON 文本 | 记录模拟原计划及必要分析字段 |
| original_amount | 数值 | 修改前模拟金额 |
| conflicts_json | JSON 文本 | 系统发现的个人规则冲突 |
| risks_acknowledged | 布尔 | 用户是否认可新发现的风险 |
| final_choice | 枚举 | 维持原计划、修改计划、稍后再看 |
| revised_amount | 数值/空 | 修改后模拟金额 |
| duration_seconds | 数值 | 从开始测试到提交反馈的时间 |
| satisfaction | 1–5 整数 | 单次满意度 |
| repeat_intent | 布尔 | 是否愿意下次再次使用 |
| paid_test_intent | 布尔 | 是否愿意了解小额付费测试 |
| notes | 文本 | 一句话匿名反馈 |
| created_at | 时间 | 本地保存时间 |

## 明确不收集

姓名、身份证号、手机号、微信号、券商账户、券商密码、银行卡号、短信验证码和精确真实资产。测试界面要求使用模拟值，备注框也提醒不要填写身份信息。

## 保存、导出与删除

数据保存在运行环境的 SQLite `user_tests` 表中，可从“用户测试”页导出 UTF-8 CSV，并可一键删除全部测试记录。Streamlit Community Cloud 本地磁盘可能重启丢失，因此每轮访谈结束后应立即导出，保存在受控研究文件夹中；不要上传公开仓库。

## 分析口径

首轮只统计任务完成数、完成时间、风险认可率、计划改变率、满意度、复用意愿和付费测试意愿。样本量小，不把百分比当作市场规模结论；结合访谈原话判断原因。
