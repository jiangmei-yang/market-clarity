# 微信小程序原型架构

## 当前定位

`miniprogram_prototype/` 是课程体验原型，不替代移动网页，也不代表已经备案或审核通过。默认使用模拟响应，可以在微信开发者工具中展示完整路径。

## 页面

```text
欢迎
  → 自然语言设置
  → 规则确认
  → 五项交易输入
  → 理由结构确认
  → 第二意见卡
  → 修改前后对比
```

不包含完整行情、自选、财务图表、社区、券商连接、下单或微信支付。

## 前后端边界

小程序只负责输入、确认和展示。FastAPI 负责：

- 结构化 JSON 解析；
- 股票代码解析与行业资料；
- 受控知识检索；
- 仓位、集中度和损失情景计算；
- OpenAI 调用；
- 未来的身份、数据库和删除接口。

小程序通过 HTTPS 调用 `/v1/onboarding/parse`、`/v1/decision/parse`、`/v1/decision/review`。当前接口未实现正式用户鉴权，不可直接作为公开生产 API。

## 密钥边界

前端只能保存公开 API 基址。以下内容只能存在于服务器 Secrets 或环境变量：

- `OPENAI_API_KEY`
- `TUSHARE_TOKEN`
- 微信小程序 `AppSecret`
- 数据库管理凭据

正式接入微信登录时，由后端使用临时登录凭证换取会话；AppSecret 不下发到前端。日志必须过滤 Token、完整交易理由和身份标识。

## 生产化缺口

- 用户身份、会话过期、鉴权和限速；
- API `/v1` 的错误契约与幂等性；
- 正式数据库和逐用户删除；
- HTTPS 域名、备案与小程序服务器域名配置；
- 隐私同意、撤回和数据主体请求；
- 内容安全与人工升级流程；
- 审核通过的服务类目和产品文案。
