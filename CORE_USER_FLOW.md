# 网页版核心用户流程

## 首次使用（目标：2 分钟）

```text
首页
→ 用普通中文描述资金和提醒边界
→ 系统显示“原话 → 结构化规则”
→ 用户逐项修改或选择提醒模板
→ 用户确认并保存
```

这里的关键控制点是“确认”：AI 只负责理解语言，不能替用户决定合适仓位。

## 每次检查（目标：60 秒）

```text
开始决策审查
→ 输入股票 / 操作 / 金额 / 理由 / 失效条件
→ 确认事实、推断、未核实信息、来源和期限
→ 有限资料核实 + 确定性金额计算
→ 阅读一张审查卡
→ 维持 / 修改 / 稍后再看
→ 保存前后差异
```

阅读顺序固定为：计划与数字影响 → 理由结构 → 有限证据 → 个人规则冲突 → 用户选择。系统始终避免“可以买”“应该卖”等结论。

## 用户测试支线

匿名测试页生成 `T001` 类编号并开始计时，自动载入固定演示。完成审查后记录用户是否发现新风险、最终选择、修改金额、用时、满意度、复用意愿和付费测试意愿；研究人员在同一页面导出 CSV。

## 异常和边界

- 无 AI Key：使用本地结构化规则，不阻断流程；
- 在线数据失败：使用固定演示数据并标注状态；
- AI 理解错误：用户在计算前逐项纠正；
- 有限资料未找到证据：只表述“有限资料中未找到”，不声称证伪；
- 出现明确自伤危险表达：停止股票审查、不保存该笔内容，显示即时专业支持信息；应用不自动联系家人或第三方。
