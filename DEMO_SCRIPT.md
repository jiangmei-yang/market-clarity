# 90 秒课堂演示脚本

## 演示前

- 打开网页版首页，确认演示模式可用；
- 不依赖 OpenAI Key、实时行情或网络数据；
- 使用默认个人规则：总资金 20 万元、单股上限 25%、单笔上限 3 万元；
- 不使用任何真实账户或真实资产。

## 演示流程

**0–10 秒：入口**  
在首页点击“90 秒演示”，直接进入固定案例。

**10–25 秒：五项计划**  
页面已填入宁德时代、补仓 50,000 元，以及理由：“已经跌了很多，朋友说公司拿到了大订单，应该快反弹了。”失效条件留空。点击“整理这笔计划”。

**25–40 秒：确认理解**  
指出系统拆出的三类内容：可核实的价格描述、朋友转述的大订单、关于反弹的推断。展示来源和失效条件缺失，点击确认。

**40–65 秒：审查卡**  
指出固定演示公告标题中没有找到大订单支持，同时强调演示资料不能证伪传闻；规则引擎显示原持仓 34,000 元加计划 50,000 元后，单股占总资金 42%，超过用户自己的 25% 提醒线；若该仓位再跌 20%，金额影响为 16,800 元。

**65–82 秒：修改计划**  
点击“修改计划”，将金额改为 10,000 元。修改后单股金额为 44,000 元，占总资金 22%。展示修改前后规则冲突数量的变化。

**82–90 秒：结论**  
说明：“系统没有说该买还是该卖，只让用户看到理由中哪些尚未核实、金额是否越过自己的边界，并记录决定是否改变。”

## 失败预案

若在线数据、AI 或 Token 不可用，继续使用内置演示；若浏览器刷新，返回首页重新点击演示。不要临场切换小程序、Android 或外部行情页面。
