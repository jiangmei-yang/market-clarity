# 安心看股产品评审

评审日期：2026-07-21  
评审对象：第一版可运行 Demo  
评审视角：首次使用、非专业、偏中老年的 A 股个人投资者

## Executive verdict

第一版证明了“能做”，还没有证明“有人持续需要”。原界面更像一个功能完整但信息较少的行情终端：用户先看到指数、价格、K 线和指标，真正差异化的通俗总结与可解释风险在个股页最底部。若按原顺序测试，用户很可能把它理解成“简化版东方财富”，而不是“决策前冷静检查”。

本轮应验证的核心假设只有一个：**当普通投资者准备关注或操作一只股票时，一页有证据的中文摘要能否让他更快发现遗漏风险，并更愿意写下自己的判断。**

## Current Strengths

- 风险规则透明：有触发原因、证据、数据日期和规则编号，不把模型意见伪装成事实。
- 演示数据、空数据和接口失败都有边界提示，可信度设计优于一般原型。
- 中文、大字体、颜色加文字，方向上适合中老年用户。
- 自选、持仓、交易复盘形成了“看—想—记”的闭环雏形。
- 不连接券商、不自动下单、不承诺收益，产品边界清楚。

## Current Weaknesses

- 原首页首屏只有指数和模块，没有一句能让用户复述的价值主张。
- 搜索入口不在首页，最重要任务必须先理解侧边栏。
- 原个股页顺序是“数据 → 图表 → 财务 → 风险 → 总结”，与普通人的阅读决策顺序相反。
- 七个侧边页面同时出现，首次用户不知道从哪里开始。
- “AI 总结”尚未真正接入；如果用 AI 字样包装规则摘要，会伤害信任。
- 自选、持仓、交易日志都是合理功能，但尚无真实行为数据证明用户需要频繁使用。
- 模拟数据价格看起来很真实，虽有警告，课堂演示仍可能被误认成当前行情。

## Critical UX Problems

### P0

1. **10 秒内不知道产品是什么。** 原首页没有“给谁、解决什么问题、与行情软件有什么不同”。
2. **核心价值埋在页面底部。** 用户必须穿过大量指标才能看到通俗总结和风险。
3. **没有清晰主任务。** 首页没有“查一只股票”的主按钮，侧边栏承担了产品引导。
4. **差异化不成立。** 如果只是 K 线、MA、RSI、财务，东方财富/同花顺更全、更快。
5. **AI 命名风险。** 当前摘要是规则生成，不能声称是 AI；评委一问数据来源就会暴露叙事不一致。

### P1

1. 风险列表把“未触发”与“已触发”同等铺开，增加阅读成本。
2. 基础信息卡片过多，市盈率/市净率对普通用户缺少“所以呢”的解释。
3. “设置”“数据和隐私”对首次体验不是主任务，但在导航中与个股分析同权重。
4. 交易日志字段多，首次记录成本高，可能造成放弃。
5. 没有行为埋点，无法知道用户是否看到总结、是否理解风险、是否完成任务。

### P2

1. 涨跌颜色设置尚未完全作用到所有图表。
2. 首页指数对核心任务贡献低，可折叠。
3. 缺少真实公告会限制“决定前核实”的可执行性，但不应在验证前急着开发。

## Must Fix Before User Testing

### P0（本轮已执行）

- 首页首屏明确价值主张，并提供唯一主动作“开始检查”。
- 首页直接回答“为什么不用东方财富”和“为什么不用 ChatGPT”。
- 个股页改为“一分钟总结 → 风险 → 走势 → 公司经营 → 下一步”。
- 只在首屏突出已触发风险，完整规则放入折叠区。
- 加入一键贵州茅台 Demo，确保两分钟内进入价值场景。
- 明确写出“摘要由固定规则整理，不依赖大模型”。

### P1（测试前补轻量测量，不扩功能）

- 记录匿名事件：开始检查、看到风险、加入自选、记录判断；不记录持仓内容。
- 将首次交易日志缩减为 3 个必填项，其余字段渐进展开。
- 为用户测试准备真实但脱敏的 3 只股票场景，避免只测贵州茅台。

## Nice To Have

### P2

- 公告原文链接与“需要核实什么”形成闭环。
- 可打印的一页检查报告。
- 家属协助模式：大字版和远程共享摘要。
- 仅当用户研究证明摘要价值后，再测试 AI 改写与反方提问。

## Pages: keep, merge, or remove?

| 页面 | 决策 | 原因 |
|---|---|---|
| 首页 | 保留并重做 | 承担价值说明和唯一入口 |
| 个股分析 | 保留，改名“个股检查” | 核心价值页面 |
| 自选股 | 保留但降级 | 可能支持回访，需验证 |
| 持仓 | 保留但不作为 Demo 主线 | 价值潜力高，但输入成本高 |
| 交易日志 | 保留，测试使用率 | 行为改变的关键假设 |
| 设置 | 导航降级候选 | 最终用户很少需要 |
| 数据和隐私 | 保留 | 金融产品建立信任所必需 |

## 最重要的非功能决定

未来两周不开发新闻、自动交易、预测模型、复杂 AI、同行数据库或社交功能。只有真实用户重复出现同一个障碍，且界面文案/流程无法解决时，才进入开发队列。
