# 安心看股 MVP 完成报告

**状态日期：2026-07-21**  
**交付平台：Streamlit 网页版**

## 已完成的核心闭环

1. 自然语言设置个人提醒边界，结构化后由用户确认；
2. 用一句话输入股票、操作、金额和理由，并允许逐项纠正；
3. 将理由拆成可核实事实、未核实外部信息、主观推断和情绪表达；
4. 用确定性代码计算操作后仓位、行业集中度和 10%/20%/30% 下跌情景；
5. 从受控知识库返回风险解释，并检索近 180 日公告标题核对传闻；
6. 生成一屏式决策审查卡，用户选择保持、修改或稍后再看；
7. 保存修改前后的金额、单股仓位、20% 下跌影响和规则提醒数；
8. 决策历史、匿名测试记录、持仓 CSV 导入/导出、访问密码和数据隐私页均已可用。

## 90 秒固定演示验收

- 首页有独立“90 秒演示”入口；
- 演示强制使用固定规则、持仓和资料，不需要 OpenAI Key 或在线行情；
- 50,000 元补仓后显示单股仓位 42%，若再跌 20% 的金额影响为 16,800 元；
- 修改为 10,000 元后显示单股仓位 22%，金额影响为 8,800 元；
- 页面不输出“买入/卖出”结论，也不会自动执行交易。

## 数据与降级策略

`AKShare 真实公开数据 → 最近缓存 → 明确标记的演示资料`

- 无 OpenAI Key：本地规则解析理由，核心流程不中断；
- 无 Tushare Token：使用 AKShare；
- 公告标题命中：只显示“可能相关披露”并提供原文链接；
- 公告标题未命中：明确说明“未找到不等于已证伪”。

## 工程验收

- 55 项自动化测试通过；
- 独立 Streamlit 进程可正常启动；
- 360×800、390×844、412×915 三种手机尺寸均无页面横向溢出；
- 手机核心按钮可见高度为 45–48px，主要决策按钮为 48px；
- 真实 Secrets 文件、SQLite、缓存和虚拟环境已被 Git 忽略；
- 代码中未发现真实 API Key、Token 或密码；
- Streamlit Cloud 依赖与开发/测试依赖已拆分。

## 已知限制

- Community Cloud 本地 SQLite 不是永久存储，需定期导出 CSV；
- AKShare 为第三方公开接口，可能受上游字段和限流影响；
- 当前公告核实只比对标题关键词，不做全文语义判定；
- Streamlit 不是完整离线 PWA，当前交付是可添加到桌面的手机网页。

## 不在本目标内

真实用户的任务完成率、复用意愿、付费意愿和行为改变证据尚未采集。产品基线完成后，下一阶段应按 `USER_TEST_PLAN.md` 邀请 5–10 位目标用户。
